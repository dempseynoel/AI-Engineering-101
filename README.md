# AI Engineering Training Programme

**April – October 2026 | Capgemini Invent**

A six-month programme designed to upskill our teams in AI engineering, taking participants from cloud and containerisation fundamentals through to building and deploying production-grade AI systems.

## What This Repo Contains

This repository is the central home for all labs, case studies, and supporting materials produced throughout the programme. Each month has its own directory containing hands-on exercises, reference code, and documentation.

| Month | Theme | Summary |
|-------|-------|---------|
| **April** | Foundations | Cloud setup, Docker, AI/ML app deployed end to end |
| **May** | RAG Engineering | Retrieval pipelines, vector stores, evaluation and observability |
| **June** | Agentic Systems | Tool-using agents, memory, multi-agent patterns, guardrails |
| **July** | Productionisation | Managed compute, identity, secrets, monitoring, red-teaming |
| **August** | Buffer | Catch-up, exam prep, and hackathon planning |
| **Sep – Oct** | Hackathon | Team build sprint followed by showcase and review |

## Who Is This For

- **Hands-on track (AC → MC):** Engineers building, deploying, and operating AI systems across Azure, AWS, GCP, and Databricks.
- **Conceptual track (SM+):** Senior managers focusing on architecture decisions, cost modelling, governance, and vendor strategy.

## How the Programme Works

Each month (April–July) follows the same rhythm: a vendor-led demonstration, a hands-on lab, a Capgemini case study grounded in realistic client scenarios, and a small homework assignment. One in-office day per month is protected for learning. Everything is designed to fit within office hours.

The programme culminates in a two-month hackathon (September–October) where small pods build end-to-end AI solutions and present them to a mixed vendor panel.

## Certifications

The programme aligns with industry credentials so that learning time counts towards formal recognition. Participants choose a primary certification based on their project context:

- Azure AI Engineer Associate
- Databricks Generative AI Engineer Associate
- Google Cloud Generative AI Leader
- AWS Machine Learning pathways

## Getting Started

Each lab folder contains its own README with prerequisites, setup instructions, and exercises. Start with the April lab and work through in order.
